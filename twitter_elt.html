Project Description

Objective

The aim of this project is to develop an ELT (extract, load, transform) pipeline on Twitter tweets from @WSJ to preodically process and email the top tweets of the day according to retweet count. I used the tweepy library to extract the tweets, MongoDB to store them, the smtplib library to send e-mails, and Apache Airflow for job scheduling.

Methodology

Extract: Firstly, I used the tweepy library to extract tweets from Wall Street Journal's Twitter account.

Load: I stored the tweets into a MongoDB database, with features such as the timestamp that the tweet was created, the number of retweets, and the tweet text content.

Transform: Using MongoDB queries, I retrieved the top three tweets with the highest retweet count, and used the smtplib module to send an automated e-mail containing the top three tweets for the day.

Job Scheduling: Finally, I used Apache Airflow to perform job scheduling on the ELT pipeline, allowing the automated email to be sent daily.
